Before You Write In

Most "how do I…" questions are covered in the FAQ below. For bug reports, please include:

  • macOS version (About This Mac → More Info → macOS).
  • Terminal Conductor SSH version (Terminal Conductor SSH menu → About).
  • What you did, what you expected, what happened instead.
  • Any red entries in the Console tab of View → Toggle Developer Tools.
  • If the App crashed: the most recent crash log from ~/Library/Logs/DiagnosticReports/.

Frequently Asked Questions

General

What is Terminal Conductor SSH?

Terminal Conductor SSH is a macOS application that wraps your remote SSH sessions with identity, context, memory, and orchestration. It's built for developers running multiple AI coding agents alongside remote shells over SSH.

Does it replace my remote shell or dotfiles?

No. Terminal Conductor SSH starts your normal login shell on the remote (zsh, bash, fish — whatever your $SHELL is there). Your aliases, prompts, plugins, and tools behave exactly as they always have.

What macOS versions are supported?

macOS 14.0 (Sonoma) or later. Apple Silicon and Intel Macs are both supported.

Is any data sent to the cloud?

No. Terminal Conductor SSH has zero telemetry and no external servers. All state lives locally. See the Privacy Policy for the full breakdown.

Tabs, groups, and workspaces

How do I create a tab?

Click the + button in the tab bar, or press ⌘T. A prompt appears inside the tab: Connect to ssh user@host:. Click the caret next to + to pick a Pinned or Recent connection instead.

How do I group tabs?

Right-click a tab → Add to group → pick an existing group or create a new one. Grouped tabs render with a colored bar above them and can be acted on together (broadcast, briefing, etc.).

How do I split a tab into multiple columns?

Right-click the tab → Split view → choose 2–5 columns. Press Esc to collapse.

Tabs are auto-named "Tab A", "Tab B" — how do I rename?

Click the tab's color dot or long-press the tab; a label editor appears. Once a label is set, it takes priority over the auto-name.

AI agents

Which AI agents does Terminal Conductor SSH support?

Any that run in a terminal on the remote. Auto-detection is built for Claude Code, Codex CLI, Gemini CLI, and Aider, including model-name recognition where the agent exposes it. Other CLI-based agents still work — you just won't get the badge.

What is @tab:?

Type @tab: in any terminal to open a picker listing other tabs. Select one and Terminal Conductor SSH injects a formatted context block — identity, recent output, activity — into your current prompt. @group: does the same for whole groups.

How does /fork work?

In any agent tab, type /fork. Terminal Conductor SSH captures the resume command, creates a paired child tab with the same SSH login, and runs the resume command in both — you now have two tabs resuming the same conversation, diverging from there. Works for Claude Code and Codex out of the box; for Gemini we built our own fork flow because the official CLI didn't ship one.

Where do I put my custom slash-commands?

Open Settings → Agent Integration → Commands & agents folders and click Claude commands, Codex prompts, Gemini commands, or Claude agents. Finder opens the folder; drop .md files in and they appear in the right panel. On connect, Terminal Conductor SSH uploads those folders to the remote so your commands work there too.

My custom slash-command isn't recognized on the remote. What's wrong?

Make sure the agent was restarted after connecting (some agents load commands once at startup). If it's still unrecognized, re-open Settings → Agent Integration → Commands & agents folders and confirm the .md file is actually there — the folder on first launch only contains a single welcome.md example.

Why does Codex show my command as /prompts:NAME/?

That's Codex's required slash-command syntax — leading /prompts: and a trailing /. Terminal Conductor SSH renders Codex commands in that format in the right panel so clicking them types the correct string.

Sessions and persistence

What happens when I quit the App?

Each SSH tab is wrapped in a named tmux session on the remote. Quitting the App disconnects the SSH client but leaves the tmux session running on the remote host. Relaunching Terminal Conductor SSH reconnects and reattaches — your agent conversations and shell state are preserved exactly.

What if the remote doesn't have tmux?

Tabs still work — they fall through to a plain shell — but they can't persist across App restarts. Install tmux on the remote for full persistence. brew install tmux on macOS, apt install tmux on Debian/Ubuntu, yum install tmux on RHEL/CentOS.

I accidentally closed a tab — can I get it back?

Yes. Clicking × on a tab detaches rather than kills it. Click the caret next to the + button to see detached sessions and reattach — same remote, same tmux session, same agent conversation.

How do I permanently terminate a session?

Click the red trash icon next to Restart in the identity bar. Confirm by clicking again within a few seconds. The remote tmux session is killed, the local database row is removed, and a fresh Connect prompt appears.

Connections

How do I connect to an SSH host?

Click + to open a new tab. Type user@host at the Connect to ssh user@host: prompt and press Enter. If the host isn't in your saved credentials, you'll be prompted for the password inline — no popup, no modal.

How do I pin an SSH host?

Click the caret next to the + button → Recent tab → hover a row → click the pin icon. Pinned entries never evict from the list.

Does Terminal Conductor SSH type my SSH passwords?

Only if you enable that feature. Add an entry for user@host under Settings → Terminal → Saved SSH credentials, enter the password once, and Terminal Conductor SSH will use it the next time you connect to that host. Passwords are stored with light obfuscation inside the App's sandbox container — this is convenience, not security. Use keys for anything that matters.

My SSH connection drops after a while. Why?

Terminal Conductor SSH sets ServerAliveInterval automatically and sends a null-byte keepalive ping, so most NAT and firewall idle drops are prevented. If your network is especially aggressive, adjust the ping frequency in Settings → Terminal → Connection.

Remote environment

What shows up in the identity bar for an SSH tab?

Tab name, Node version, Python version (if installed), virtualenv name (if active), git branch and dirty state, detected framework (Next, Vite, Rails, Django, Express, and more), safety badge (DEV / STAGING / PROD / ROOT / SSH), and the remote shell's PID. All of this is collected by a remote probe that runs on connect.

How does the PROD badge get set?

If any environment variable on the remote contains "prod" (case-insensitive — NODE_ENV, RAILS_ENV, APP_ENV, ENVIRONMENT, DJANGO_SETTINGS_MODULE, FLASK_ENV), the tab is flagged PROD. Logging in as root triggers ROOT. "stag" triggers STAGING, "dev" triggers DEV, and everything else defaults to SSH.

The GitHub PR row is empty. Why?

It requires gh to be installed and authenticated on the remote host (not your Mac). Run gh auth login there. The row scopes to the first git repo it finds in your remote home directory.

Visual modes and borders

How do I change visual mode or border?

Open the right panel → Appearance → Visual Mode / Border Decoration. Settings are per-tab by default; use groups to apply to several at once.

How do I turn off the JARVIS HUD border?

The JARVIS HUD is bound to the JARVISmaxxing visual mode. Switch to a different visual mode and the border changes with it. Or turn off Force mode borders in Settings → Appearance to decouple mode and border.

Can I use the 1992 skin on a modern macOS?

Yes — it's purely cosmetic. All functionality is identical. Turn it on via Settings → Appearance → Skin.

Privacy and data

Where is my data stored?

Inside the App's sandbox container: ~/Library/Containers/com.terminalconductor.ssh/Data/. Session database, saved credentials (if any), settings, scrollback — all local. Nothing leaves your Mac.

How do I wipe all Terminal Conductor SSH data?

Quit the App, then delete ~/Library/Containers/com.terminalconductor.ssh/. Reinstalling gives you a clean slate. See the Privacy Policy, Section 7, for the full step-by-step.

Are passwords ever saved to shell history or scrollback?

No. A two-phase password scrubber drops password-prompt responses before they reach the Recent list, activity events, or scrollback. Passwords entered in the in-terminal SSH auth prompt go directly to the SSH library without ever touching the terminal output buffer.
