FAQ

General

What is Terminal Conductor SSH?

A macOS application that lets you manage many SSH sessions as named, color-coded, persistent tabs in a single window. Built for developers who run AI coding agents (Claude Code, Codex CLI, Gemini CLI) on remote hosts.

What macOS versions are supported?

macOS 14.0 (Sonoma) or later. Apple Silicon and Intel Macs are both supported.

Does it replace my remote shell or dotfiles?

No. It launches your normal login shell on the remote (zsh, bash, fish — whatever your $SHELL is there). Your aliases, prompts, plugins, and tools behave exactly as they always have.

Is any data sent to the cloud?

No. Zero telemetry, no external servers. All state — session database, credentials (if saved), settings — lives locally in your App Support container. See the Privacy Policy for the full breakdown.

Connecting

How do I connect to an SSH host?

Click the + button to open a new tab. A prompt appears inside the terminal: Connect to ssh user@host:. Type user@host and press Enter. If the host isn't in your saved credentials, you'll be prompted for the password inline — no popup, no modal.

How do I save a password for a host?

Open Settings → Terminal. Under Saved SSH credentials, add an entry with user@host and the password. Next time you connect to that host, Terminal Conductor SSH skips the password prompt entirely.

How are saved passwords stored?

Locally, in the app's sandbox container, with light obfuscation. This is convenience, not a security guarantee. For anything that matters, use SSH keys.

How do I pin a host?

Click the caret next to the + button → Recent tab → hover a row → click the pin icon. Pinned entries never evict.

My SSH connection drops after a while. Why?

Terminal Conductor SSH sets ServerAliveInterval automatically and sends a null-byte keepalive ping, so most NAT/firewall idle drops are prevented. If your network is especially aggressive, increase the ping frequency in Settings → Terminal → Connection.

AI agents

Which agents are auto-detected?

Claude Code, Codex CLI, Gemini CLI, and Aider. Detection is based on the agent's terminal output, so it works as long as the agent actually runs on the remote. Other CLI-based agents work too — you just won't get the color gradient and agent badge.

What is @tab:?

Type @tab: in any terminal to open a picker listing your other tabs. Select one and Terminal Conductor SSH injects a formatted context block — identity, recent output, activity — into your current prompt. @group: does the same for whole groups.

How does /fork work?

Type /fork in any Claude, Codex, or Gemini tab. Terminal Conductor SSH captures the resume command, opens a new tab with the same SSH login, and runs that resume command in both — you now have two tabs resuming the same conversation, diverging from there.

My custom slash-command isn't recognized on the remote. What's wrong?

Custom commands live in ~/.claude/commands/, ~/.codex/prompts/, or ~/.gemini/commands/. Terminal Conductor SSH uploads these to the remote on every connect. If a command still isn't recognized: make sure the agent is fully restarted after connecting (some agents load commands once at startup), and check that the files exist in the expected local paths on your Mac.

Why does Codex show my command as /prompts:NAME/?

That's Codex's required slash-command syntax — leading /prompts: and a trailing /. Terminal Conductor SSH renders Codex commands in that format in the right panel so clicking them types the correct string.

Session persistence

What happens when I quit the app?

Each SSH tab runs inside a named tmux session on the remote. Quitting the app disconnects the SSH client but leaves tmux running on the remote. Relaunching reconnects and reattaches — your agent conversations and shell state are preserved exactly.

What if the remote doesn't have tmux?

Tabs still work — they fall through to a plain shell — but they can't persist across app restarts. Install tmux on the remote for full persistence. brew install tmux on macOS, apt install tmux on Debian/Ubuntu, yum install tmux on RHEL/CentOS.

I closed a tab by accident — can I reattach?

Yes. Clicking × on a tab detaches rather than kills it. Click the caret next to the + button; detached sessions appear in the dropdown. Click to reattach — same remote, same tmux session, same agent conversation.

How do I permanently terminate a session?

Click the red trash icon next to Restart in the identity bar. Confirm by clicking again within a few seconds. The remote tmux session is killed, the database row is removed, and a fresh Connect prompt appears.

Remote environment

What shows up in the identity bar for an SSH tab?

Tab name, Node version, Python version (if installed), virtualenv name (if active), git branch and dirty state, detected framework (Next, Vite, Rails, Django, Express, and more), safety badge (DEV / STAGING / PROD / ROOT / SSH), and the remote shell's PID. All of this is collected by a remote probe that runs on connect.

How does the PROD badge get set?

If any environment variable on the remote contains "prod" (case-insensitive — NODE_ENV, RAILS_ENV, APP_ENV, ENVIRONMENT, DJANGO_SETTINGS_MODULE, FLASK_ENV), the tab is flagged PROD. Logging in as root triggers ROOT. "stag" triggers STAGING, "dev" triggers DEV, and everything else defaults to SSH.

What's in the bottom panel for an SSH tab?

Summary of recent activity, test results (if the agent ran tests), env-delta, an in-flight process strip showing the remote foreground + services, and a GitHub PR row (if gh is authenticated on the remote).

The GitHub PR row is empty. Why?

It requires gh to be installed and authenticated on the remote host (not your Mac). Run gh auth login there. The row scopes to the first git repo it finds in your remote home directory.

Visual modes and appearance

How do I change the visual mode?

Open the right panel → Appearance → Visual Mode. Settings are per-tab by default — you can make all SSH tabs in a group look the same by changing the group's default, or leave them independent.

Can I use the 1992 skin on modern macOS?

Yes. It's purely cosmetic — no behavior differences. Turn it on via Settings → Appearance → Skin.

Privacy and data

Where is my data stored?

Inside the app's sandbox container: ~/Library/Containers/com.terminalconductor.ssh/Data/. Session database, saved credentials, settings, scrollback — all local. Nothing leaves your Mac.

How do I wipe all data?

Quit the app, then delete ~/Library/Containers/com.terminalconductor.ssh/. Reinstalling gives you a clean slate.

Are passwords ever saved to shell history or scrollback?

No. A two-phase password scrubber drops password-prompt responses before they ever reach the Recent list, activity events, or scrollback. Additionally, passwords entered in the in-terminal SSH auth prompt go directly to the SSH library without touching the terminal output buffer.
